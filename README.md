# SpotCast — Мобильное приложение для создания гео-капсул

Проект представляет собой Android-приложение для обмена гео-привязанными текстовыми и аудио сообщениями с использованием фонового Geofencing-отслеживания и системы слоев.
Для развертывания и полноценной работы приложения потребуется собственный VPS сервер. В исходном коде данные к нему скрыты в целях безопасности.
## Требования к окружению

### На стороне сервера (Backend):
1. **Python 3.10+**.
2. Установленная СУБД **PostgreSQL** (версии 13+) с обязательным активированным расширением **PostGIS** (для работы с географическими полигонами и координатами).
3. Инструмент пакетной установки `pip`.

### На стороне клиента (Android):
1. **Android Studio** (последняя стабильная версия).
2. **JDK 17**.
3. Устройство или эмулятор под управлением ОС Android 8.0+ с активными Google Play Services (обязательно для работы Location API).

## Инструкция по сборке и запуску

### Шаг 1. Развертывание Серверной части (FastAPI)

1. Перейдите в директорию `backend`:
   ```bash
   cd backend
   ```
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для Linux/macOS:
   source venv/bin/activate
   ```
3. Установите необходимые зависимости из файла `requirements.txt`:
   ```bash
   pip install -r requirements.txt
   ```
4. **Настройка базы данных:**
   Создайте в папке `backend` файл конфигурации окружения `.env` и укажите строку подключения к вашей PostgreSQL базе данных (должно быть включено расширение `postgis`). Рекомендуется использовать облачные решения, например, Supabase.
   
   Пример содержимого `.env`:
   ```env
   DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/spotcast
   JWT_SECRET=your_super_secret_key_here
   ```
   *Примечание: Если вы запускаете локальный Postgres, убедитесь, что выполнили SQL-команду `CREATE EXTENSION IF NOT EXISTS postgis;` в вашей БД.*

5. Запустите сервер:
   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8080
   ```
   API будет доступно по адресу: http://localhost:8080. Документация (Swagger) — http://localhost:8080/docs.

### Шаг 2. Развертывание Android-клиента

1. Откройте проект в **Android Studio**, выбрав корневую директорию `spotcast_public`.
2. Подождите завершения процесса синхронизации Gradle (Gradle Sync).
3. Перейдите в файл конфигурации сети по пути:
   `app/src/main/java/com/example/spotcast/data/remote/RetrofitClient.kt`
4. Найдите строку константы `BASE_URL`:
   ```kotlin
   private const val BASE_URL = "http://YOUR_SERVER_IP:8080/"
   ```
   Замените `YOUR_SERVER_IP` на фактический IP-адрес вашего сервера (если запуск происходит локально, а тестирование через эмулятор — используйте `http://10.0.2.2:8080/`).
5. Выберите устройство/эмулятор сборки сверху на панели инструментов и нажмите "Run" или выполните команду сборки APK через терминал:
   ```bash
   ./gradlew clean assembleDebug
   ```
6. Скомпилированный установочный файл (apk) будет доступен по пути:
   `app/build/outputs/apk/debug/app-debug.apk`

## Архитектура
* **Клиент:** Kotlin, Jetpack Compose, архитектура MVVM, Room Database, Retrofit, аппаратный Google Geofencing.
* **Сервер:** FastAPI, SQLAlchemy (asyncpg), PostgreSQL + PostGIS (ST_DWithin).

## Авторизация
Токены авторизации базируются на JWT с алгоритмом подписи HS256. В целях тестирования регистрация новых пользователей открыта напрямую через эндпоинт `/auth/register` внутри приложения.
